\begin{abstract}
Standard Transformer attention has $O(N^2)$ \emph{memory traffic} when the $N \times N$ score matrix is materialized in HBM. This project implements a FlashAttention-style forward pass in bare CUDA for an NVIDIA Tesla T4 (Turing, sm\_75), starting from a three-kernel naive baseline and adding a sequence of increasingly hardware-aware variants: shared-memory tiling with online softmax, causal masking for autoregressive attention, a warp-shuffle \mbox{row-wise} implementation, a Tensor Core (WMMA) matmul path with FP32 softmax state, and a double-buffering entry point that (on sm\_75) documents the lack of a separate async-copy win versus the WMMA kernel. Measurements combine \texttt{cudaEvent} latency sweeps and Nsight Compute global-byte counters; RMSE checks tie custom kernels to naive references.
\end{abstract}

\section{Introduction and Motivation}
Transformer inference is often bandwidth-bound: moving partial attention tensors through HBM wastes energy and time compared with compute~\cite{dao2022}. Implementing FlashAttention from scratch clarifies how tiling, softmax fusion, and structured reductions interact with the memory hierarchy on commodity GPUs~\cite{dao2022,milakov2018}.

\section{Hardware and Experimental Setup}
Experiments target an NVIDIA Tesla T4 (Turing, sm\_75), CUDA via \texttt{nvcc}, CMake Release builds (\texttt{-O3}, \texttt{--extended-lambda}, \texttt{--expt-relaxed-constexpr}, \texttt{-lineinfo}). Timing uses GPU events with warmup iterations; HBM traffic aggregates NCU \texttt{l1tex\_\_t\_bytes\_pipe\_lsu\_mem\_global\_op\_\{ld,st\}.sum} metrics. Correctness compares each optimized kernel against an appropriate naive baseline (bidirectional or causal) using RMSE; FP16 Tensor Core paths allow RMSE up to $5 \times 10^{-3}$ versus FP32 references.

\section{Baselines and Implementations}

\subsection{Naive Three-Kernel Attention}
We compute $S = Q K^\top / \sqrt{d}$, apply row softmax, then $O = \mathrm{softmax}(S) V$ as separate kernels. For causal experiments we mask forbidden positions with large negative logits before softmax (\texttt{naive\_attention\_causal}).

\subsection{FlashAttention v1 (Tiled FP32)}
\texttt{flash\_attention\_v1.cu} tiles $Q$, $K$, and $V$ in shared memory ($B_r=B_c=32$, $d=64$), maintains running softmax statistics $(m,\ell)$ per query row in registers, and never spills $S$ to HBM~\cite{dao2022,milakov2018}.

\subsection{Causal Masking (\texttt{flash\_causal})}
We skip entire $K/V$ column tiles once every query row in the block is ``to their left,'' and apply within-tile masking on diagonals so logits obey $j \le i$. This removes useless compute for upper-triangular regions without changing numerics where keys are valid.

\subsection{Warp-Shuffle Row Variant (\texttt{flash\_v2})}
\texttt{flash\_attention\_v2.cu} assigns one warp per query row ($B_r{=}8$, $B_c{=}32$, $256$ threads per block). Dot products reduce across lanes with \texttt{\_\_shfl\_down\_sync}; softmax normalization runs on lane~0 with weights staged through shared memory so PV accumulation stays coherent across warps.

\subsection{Tensor Core Path (\texttt{flash\_wmma})}
\texttt{flash\_attention\_v3\_wmma.cu} evaluates each $16{\times}16$ score tile using WMMA \texttt{m16n16k16} MMA on FP16 operands loaded from shared memory while softmax statistics remain FP32. Keys are packed into WMMA's column-major \texttt{matrix\_b} layout per $16$-column slice of $d{=}64$.

\subsection{``Double Buffer'' Variant (\texttt{flash\_wmma\_db})}
Ampere-style \texttt{cp.async} overlap is not assumed on Turing; \texttt{flash\_attention\_v3\_wmma\_db} currently dispatches the WMMA kernel so benchmarks quantify whether standalone pipelines materially improve throughput without rewriting kernel internals. On sm\_75 we expect parity---documented as a negative architectural result rather than a claimed speedup.

\subsection{Discussion}
\begin{itemize}
    \item \textbf{Memory vs compute:} FlashAttention removes the dominant quadratic HBM footprint; further gains hinge on instruction throughput (warp specialization, Tensor Cores).
    \item \textbf{Causal masking:} Provides nearly ``free'' flops reductions whenever sequences exceed tile width because wasted tiles never enter softmax normalization.
    \item \textbf{Turing Tensor Cores:} FP16 WMMA paths demand careful packing but unlock sustained throughput beyond FP32 scalar lanes for matmul-heavy slices.
    \item \textbf{Async copies on sm\_75:} Without Ampere-style \texttt{cp.async}, overlapping loads rarely beats careful synchronous tiling when kernels remain latency-bound on softmax bookkeeping.
\end{itemize}
